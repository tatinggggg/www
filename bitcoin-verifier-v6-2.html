🔐 Sign & Verify (ECDSA/secp256k1)

Bitcoin signing uses ECDSA on the secp256k1 curve. The private key produces two integers (r, s) by signing the double-SHA256 of the serialized transaction (SIGHASH). Nodes verify the signature against the public key in the previous output's scriptPubKey without ever exposing the private key.

How Bitcoin Transactions Work: Every Bitcoin transaction starts as raw hex — a serialized binary structure encoding inputs, outputs, amounts, and scripts. Before a node accepts it, the transaction must be signed with the sender's private key using ECDSA (secp256k1 curve). Miners then include signed transactions in blocks, and each subsequent block adds a confirmation.
